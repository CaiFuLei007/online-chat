# online-chat Docker 使用手册

本手册介绍如何使用 Docker 启动、管理和验证 online-chat 服务。

---

## 1. 整体架构

`docker compose` 一次性启动 3 个容器：

| 容器名 | 镜像 | 作用 | 宿主机端口映射 |
|--------|------|------|----------------|
| oc-app | docker-app（本地构建） | 应用服务（HTTP + WebSocket） | `8080 -> 8080` |
| oc-mysql | mysql:8.0 | 数据库（首次启动自动导入 `sql/schema.sql`） | `3307 -> 3306` |
| oc-redis | redis:7-alpine | 缓存 / 验证码存储 | `6380 -> 6379` |

> 宿主机本身已运行 MySQL（3306）和 Redis（6379），所以容器对外端口错开为 3307 / 6380。
> 容器之间通过内部网络互联（app 直接访问 `mysql:3306`、`redis:6379`），不受此影响。

数据持久化（Docker 卷，`docker compose down` 不会丢）：

| 卷名 | 内容 |
|------|------|
| mysql_data | MySQL 数据 |
| redis_data | Redis 数据 |
| uploads_data | 用户上传的文件（`/app/frontend/uploads`） |

---

## 2. 前置条件

1. 已安装 Docker 与 Docker Compose v2（本机已满足，当前用户在 docker 组，**无需 sudo**）。
2. 宿主机端口 **8080、3307、6380** 空闲。
3. **构建镜像时**需要能访问 GitHub（编译 Drogon 框架源码）。
   compose 已配置自动继承宿主机 shell 的 `http_proxy` / `https_proxy` 环境变量作为**构建期代理**（仅构建用，不进入运行时镜像）。如果你的代理地址变了，确保终端里 `echo $https_proxy` 输出正确即可。
4. 宿主机目录 `frontend/uploads/` 必须存在（作为卷挂载点，已创建）。

---

## 3. 需要配置的内容

### 3.1 .env 文件（可选，但生产必配）

项目提供了 `.env.example` 模板，复制后填入你自己的配置即可：

```bash
cd docker
cp .env.example .env
vim .env   # 编辑填入你的 JWT_SECRET 和 SMTP 配置
```

模板中已包含说明注释。关键配置项：

| 变量 | 必填 | 说明 |
|------|------|------|
| `JWT_SECRET` | ✅ | JWT 签名密钥，建议 `openssl rand -base64 32` 生成 |
| `SMTP_HOST` | ✅ | SMTP 服务器地址（如 `smtp.qq.com`、`smtp.163.com`） |
| `SMTP_PORT` | ✅ | SMTP 端口（通常 465） |
| `SMTP_USERNAME` | ✅ | 发件邮箱账号 |
| `SMTP_PASSWORD` | ✅ | SMTP 授权码（非邮箱密码） |
| `SMTP_FROM_EMAIL` | ✅ | 发件人地址（通常与 USERNAME 相同） |

不创建 `.env` 也能启动，只是 JWT 用开发默认密钥、邮件功能不可用。

### 3.2 数据库密码（已配置好，了解即可）

- MySQL 账号密码定义在 `docker/docker-compose.yml`（`online_chat` / `online_chat_pwd`，root 密码 `root_pwd_dev`）。
- `config/config.json` 中 `db_clients.passwd` 必须与之保持一致（Drogon 不支持对该字段做环境变量替换）。改其中一处时必须同步改另一处。

### 3.3 本地开发（非 Docker 环境）

如果你不用 Docker，直接在本机编译运行，需要将数据库连接地址从 Docker 服务名改为 `127.0.0.1`：

```bash
cd config
cp config.local.json.example config.local.json
# config.local.json 已被 .gitignore 排除，不会提交
```

`config.local.json` 中 `host` 已预设为 `127.0.0.1`，按需修改 `passwd` 等字段即可。
程序启动时会优先加载 `config.local.json`（如果存在），否则使用 `config.json`。

### 3.4 端口（如需修改）

改 `docker/docker-compose.yml` 中 `ports` 的**左侧**值，例如 `"9090:8080"`，然后 `docker compose up -d` 重建即可。

## 4. 启动步骤

```bash
# 1. 进入 docker 目录
cd ~/online-chat/docker

# 2.（可选）创建 .env，见上文 3.1

# 3. 构建并后台启动全部服务
#    首次构建需从源码编译 Drogon，约 10-20 分钟；之后有缓存会很快
docker compose up -d --build

# 4. 查看容器状态（三个都应为 Up / healthy）
docker compose ps

# 5. 跟踪应用日志确认无报错（Ctrl+C 退出跟踪，不影响服务）
docker compose logs -f app
```

启动顺序由 compose 自动控制：MySQL 和 Redis 通过健康检查就绪后才启动 app。

---

## 5. 验证服务

```bash
# 前端页面，应返回 200
curl -s -o /dev/null -w "%{http_code}\n" http://localhost:8080/

# 登录接口（走通 MySQL 链路），应返回 401 + "邮箱或密码错误"
curl -s -X POST http://localhost:8080/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"nobody@test.com","password":"wrong"}'
```

浏览器访问：

- 首页 / 登录注册：<http://localhost:8080/>
- 聊天页：<http://localhost:8080/chat.html>
- 管理页：<http://localhost:8080/admin.html>

更多接口用法见项目根目录 `API.md` 与 `项目接口测试-curl.md`。

## 6. 日常管理指令

```bash
cd ~/online-chat/docker        # 以下命令均在此目录执行

docker compose ps              # 查看状态
docker compose logs -f app     # 跟踪应用日志（mysql / redis 同理）
docker compose restart app     # 重启应用
docker compose stop            # 停止（保留容器）
docker compose start           # 再次启动
docker compose down            # 停止并删除容器（数据卷保留）
docker compose down -v         # ⚠️ 停止并删除数据卷：清空数据库、Redis、上传文件，
                               #    下次启动时自动重新导入 sql/schema.sql
docker compose up -d --build app   # 改了 C++ 源码 / Dockerfile 后重建应用镜像
```

### 修改不同内容后需要做什么

| 修改内容 | 操作 |
|----------|------|
| `frontend/` 前端文件 | 无需操作，目录是挂载的，刷新浏览器即可 |
| `src/` C++ 源码、`CMakeLists.txt` | `docker compose up -d --build app` |
| `config/config.json` | `docker compose up -d --build app`（配置打包在镜像内） |
| `docker/.env` | `docker compose up -d`（重建 app 容器） |
| `docker-compose.yml` | `docker compose up -d` |
| `sql/schema.sql` | 仅对全新数据库生效；要重新导入需 `docker compose down -v` 后再启动（会清空数据） |

## 7. 连接数据库 / Redis 调试

```bash
# 方式一：从宿主机连（注意端口是 3307 / 6380）
mysql -h 127.0.0.1 -P 3307 -u online_chat -ponline_chat_pwd online_chat
redis-cli -p 6380

# 方式二：进入容器内连
docker exec -it oc-mysql mysql -u online_chat -ponline_chat_pwd online_chat
docker exec -it oc-redis redis-cli
```

## 8. 常见问题（本机踩坑记录）

| 现象 | 原因 | 解决 |
|------|------|------|
| 构建时 `git clone github.com 失败 / Connection refused` | 构建容器访问不了 GitHub | 确保终端已设置 `https_proxy`，compose 会自动透传给构建过程；Dockerfile 中 clone 已带 5 次重试 |
| 构建时 `nlohmann_json` 下载失败 | jwt-cpp 尝试从 GitHub 拉取依赖 | 已修复：builder 阶段通过 apt 安装 `nlohmann-json3-dev` |
| app 日志 `No database is supported by drogon` | Drogon 编译时未找到 MySQL 客户端库 | 已修复：使用 `libmariadb-dev`（运行时 `libmariadb3`）替代 `libmysqlclient-dev` |
| 启动报 `port is already allocated / address already in use` | 宿主机 3306/6379 被本机 MySQL/Redis 占用 | 已修复：映射改为 3307/6380；如 8080 也被占，改 compose 中映射左侧值 |
| app 日志 `creating path ./frontend/uploads/... Read-only file system` | frontend 以只读方式挂载 | 已修复：`uploads_data` 卷覆盖 uploads 目录 |
| app 反复重启、连不上数据库 | `config/config.json` 的 `passwd` 与 compose 中 MySQL 密码不一致 | 两处保持一致后重建 app 镜像 |
| 登录接口返回 401"邮箱或密码错误" | 这是正常业务响应 | 说明数据库链路正常，先注册用户即可 |

## 9. 安全提醒（生产部署前必读）

- `JWT_SECRET` 必须在 `.env` 中改为强随机值，不要使用开发默认值。
- `docker-compose.yml` 中的 MySQL 密码为开发用弱口令，生产环境应修改（同时同步 `config/config.json`）。
- 3307/6380 端口映射仅为本机调试方便，生产环境建议删掉这两条 `ports` 配置，不对外暴露数据库。
